Authorization strategies

Background：HTTP is a stateless protocol, user send a url and the server returns html,but if the server is dynamic
and need to recogonize who you are.

Two solution： Session Token and JWT

Session Token

Session ID get passed to the server and use cookies to save the session ID
Session ID + Cookies

JWT(Json Web Token)

Users send whole json token which has all the information to server, server returned a something in a speical signed
format and returned to the user then the user send it back to the server and get confirmed

Comparision

Session ID as a reference token
JWT as a value token which contains all the value itself

OAuth

  1. Send authorization request to authorization server and ask your account and show permissions.
  2. return authorization grant along access token to see accoun detail to my application.
  3. send back to authorization server
  4. return access token for resource server only permitted for the information protected to my application server
  5. send access token to resource server
  6. return protected resource

Passwords are never passed from server to server in an OAuth framework
