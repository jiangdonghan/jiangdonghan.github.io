eloquentJs读书笔记8(Bug and Error)
Created 2020-04-23| Updated 2020-04-25|javascriptelequentJs笔记
Post View:

理解JavaScript中的错误并能快速定位发现bug

严格模式

在文件或函数体顶部加”use strict”

用use strict的思维去写js代码

类型

javascirpt中只在实际运行程序时考虑类型
有许多不同的约定，标注js的程序类型
eg.
// (WorldState, Array) → {direction: string, memory: Array}
或者使用TypeScript

测试

测试通常使用小标签程序的形式来验证代码的某些方面
例如，测试toUpperCase方法

Javascript

错误传播Error Propagation

函数可以返回一个特殊值来表明错误
通常为-1,null,undefined
或者可以返回将特殊值放在object里

Javascript

异常

Javascript

Error构造器来创建异常值
Throw引发异常
try,catch 异常的捕获通过将一段代码包装在一个try块中，后跟关键字catch来完成。 当try块中的代码引发异常时，将求值catch块，并将括号中的名称绑定到异常值。 在catch块结束之后，或者try块结束并且没有问题时，程序在整个try / catch语句的下面继续执行。

异常后清理

Javascript

finally修复伤害
即使finally代码在异常退出try块时运行，它也不会影响异常。finally块运行后，堆栈继续展开。

选择性捕获

当程序出现异常且异常未被捕获时，异常就会直接回退到栈顶，并由 JavaScript 环境来处理

Javascript

可以自己定义error类
这里的catch代码只会捕获InputError类型的异常，而其他类型的异常则不会在这里进行处理。如果又输入了不正确的值，那么系统会向用户准确报告错误——“绑定未定义.

断言

断言（assertions）是程序内部的检查，用于验证某个东西是它应该是的方式。 它们并不是用于处理正常操作中可能出现的情况，而是发现程序员的错误。

Javascript

不建议尝试为每种可能的不良输入编写断言。 这将是很多工作，并会产生非常杂乱的代码。 你会希望为很容易犯（或者你发现自己做过）的错误保留他们。
